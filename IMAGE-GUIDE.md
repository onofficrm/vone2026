# 이미지 최적화 가이드

새 프로젝트 납품 전·운영 중 이미지 품질·용량·SEO를 맞추기 위한 기준입니다.

## 1. 이미지 최적화가 중요한 이유

- 페이지 로딩 속도(모바일 체감 속도, SEO)
- 서버·CDN 트래픽 비용
- 레이아웃 깨짐·흐릿한 이미지로 인한 신뢰도 하락 방지

## 2. 권장 파일 형식

| 용도 | 권장 |
|------|------|
| 사진·배경 | **WebP** (지원 불가 시 JPG) |
| 투명·로고 | PNG 또는 **SVG** |
| 아이콘·단순 그래픽 | **SVG** |
| OG·SNS 공유 | JPG 또는 PNG (1200×630) |

## 3. 권장 이미지 크기 (가로 px 기준)

| 용도 | 권장 최대 |
|------|-----------|
| 메인 Hero 배경 | 1920px 이하 |
| 섹션 이미지 | 1200px 이하 |
| 카드 썸네일 | 600px 이하 |
| 게시판 썸네일 | 600px 이하 |
| 로고 | SVG 또는 투명 PNG (필요 시만 PNG) |
| OG 이미지 | **1200 × 630** |

## 4. 파일 용량 기준

| 용도 | 권장 |
|------|------|
| Hero 이미지 | **500KB 이하** |
| 일반 섹션 이미지 | **300KB 이하** |
| 썸네일 | **150KB 이하** |
| 1MB 이상 | 반드시 압축·리사이즈 후 사용 |

## 5. 파일명 규칙

- 영어·숫자·하이픈(`-`) 사용
- 한글 파일명·공백 지양
- 예: `hero-main.webp`, `service-seo-01.webp`, `logo.svg`

## 6. 빌더 이미지 적용 방법

1. 빌더 작업 시 `_BUILDER_INPUT/assets`에 임시 저장
2. 확정 후 `/img/main` 또는 `/img/common`으로 이동
3. HTML·CSS 경로를 실제 경로로 수정
4. 빌더 임시 파일은 배포 전 정리( `_BUILDER_INPUT` 배포 제외 권장)

## 7. SEO 이미지 팁

- 의미 있는 파일명 (예: `portfolio-branding-01.webp`)
- 모든 콘텐츠 이미지에 **alt** 텍스트
- 본문과 관련된 설명·캡션
- 불필요하게 큰 원본을 그대로 `<img>`에 넣지 않기
- `_site.config.php`의 `og_image` 경로 확인

## 8. 검수 체크리스트

- [ ] 깨진 이미지(404) 없음
- [ ] 모바일·태블릿 비율·크롭 정상
- [ ] 용량 기준 충족
- [ ] 파일명 규칙 준수
- [ ] OG 이미지 1200×630, 1MB 이하 권장
- [ ] WebP 미지원 환경 fallback(JPG) 필요 시 확인

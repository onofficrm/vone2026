# 최종 청소·검수 프롬프트

새 프로젝트 납품 전 Cursor에 붙여 넣어 사용할 검수용 프롬프트 모음입니다.

**공통 주의사항 (모든 프롬프트에 적용)**

- git commit, push, FTP 배포 **금지**
- 그누보드 코어(`/bbs`, `/lib`, `/adm`, `common.php`) 및 `bbs/write_update.php`, 원본 `/skin/board/basic` **수정 금지**
- 문제만 **최소 범위**로 수정
- 작업 전 **생성·수정 예정 파일 목록** 제시 후 진행
- 작업 후 **수정·생성 파일 요약** 출력

---

## 1. 샘플 문구 찾기

```
이 프로젝트에서 아직 남아 있는 샘플·플레이스홀더 문구를 찾아 목록으로 정리해 주세요.
수정은 하지 말고, 파일 경로와 해당 문구만 보여 주세요.

찾을 예시:
- 샘플 사이트
- 회사명, 대표자명
- 010-0000-0000
- admin@example.com
- 주소를 입력하세요
- 고객의 성장을 돕는
- lorem, placeholder, example.com

공통 주의: git/배포 금지, 코어 수정 금지, 수정 전 파일 목록 제시.
```

---

## 2. 민감정보 확인

```
코드·설정·문서에 노출되면 안 되는 민감정보가 있는지 검색해 주세요.
발견 시 파일 경로와 줄 번호만 보고, 값은 마스킹해 주세요. 임의 수정은 하지 마세요.

확인 항목:
- FTP/SSH 계정
- DB 접속 정보 (data/dbconfig.php 등)
- API 키, 텔레그램 봇 토큰, 웹훅 URL
- 광고 추적 ID가 하드코딩된 경우
- 실제 고객 개인정보·테스트가 아닌 실번호

공통 주의: git/배포 금지, 코어 수정 금지.
```

---

## 3. 배포 제외 파일 확인

```
운영 서버에 올리면 안 되는 파일·폴더가 포함되어 있는지 확인해 주세요.

확인 항목:
- _BUILDER_INPUT
- style-guide.php (운영 전 숨김/삭제 권장)
- 테스트 이미지, 임시 백업 (*.bak, *~)
- .DS_Store, Thumbs.db
- 로그 파일, .env

목록만 정리하고, 삭제는 사용자 확인 후에만 진행해 주세요.

공통 주의: git/배포 금지, 코어 수정 금지.
```

## 4. 링크 깨짐 확인

```
주요 CTA·메뉴·게시판 링크가 상대/절대 경로 기준으로 올바른지 점검해 주세요.

확인 항목:
- 전화 버튼 (tel:)
- 카카오톡 버튼
- 상담문의 모달·폼
- GNB 메뉴
- 게시판 링크 (notice, gallery, inquiry 등)
- /page/privacy.php
- /page/404.php, /page/inquiry-thanks.php

깨진 링크만 최소 수정. 수정 전 파일 목록 제시.

공통 주의: git/배포 금지, 코어 수정 금지.
```

## 5. SEO 최종 확인

```
SEO 관련 설정·출력을 점검해 주세요. 문제만 최소 수정.

확인 항목:
- title, description (페이지별·_site.config.php)
- canonical
- OG image, og:title
- robots (404·완료 페이지 noindex 등)
- JSON-LD (components/seo-meta.php)
- H1 1개 원칙
- 이미지 alt

공통 주의: git/배포 금지, 코어 수정 금지, 수정 전 파일 목록 제시.
```
